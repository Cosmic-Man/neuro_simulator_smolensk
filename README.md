# Нейросимулятор Смоленска

Локальный демонстрационный прототип для анализа городских показателей, сценарного моделирования FCM × ANFIS и подготовки рекомендаций заказчику. Основа расчётов — `colab/colab_kirill/Pipeline.ipynb` и JSON-правила из той же папки.

Проект намеренно упрощён:

- нет авторизации, пользователей и ролей;
- нет PostgreSQL, SQLAlchemy и миграций;
- нет Docker и Docker Compose;
- исходные и новые квартальные данные хранятся в XLSX-файлах папки `datasets_ready`;
- пользовательский сценарий можно загрузить и скачать как JSON без серверного хранения.

## Запуск в Windows PowerShell

```powershell
cd "C:\Summer School\neuro_simulator_smolensk"
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Откройте `http://127.0.0.1:8000`. Логин и пароль не требуются. API: `http://127.0.0.1:8000/docs`.

Повторный запуск:

```powershell
cd "C:\Summer School\neuro_simulator_smolensk"
.\.venv\Scripts\Activate.ps1

```

## Работа с XLSX

Интерфейс показывает файлы `*.xlsx` из `datasets_ready`. Можно выбрать активный файл, изменить существующий квартал или добавить следующую строку. Изменения записываются прямо в выбранный XLSX.

После внесения подтверждённых данных нажмите «Переобучить модель». Нулевая служебная строка не принимается как фактический квартал. Кэш модели сохраняется локально в `runtime/models` и может быть пересоздан из XLSX.

Перед обновлением приложения сделайте резервную копию рабочего XLSX. Код можно заменить отдельно, после чего вернуть файл в `datasets_ready`.

## Сценарии

Готовые сценарии доступны сразу. Воздействия ограничены диапазоном `[-1, 1]`. Загруженный JSON остаётся только в памяти страницы. Для хранения между запусками используйте «Выгрузить JSON» и «Загрузить JSON».
