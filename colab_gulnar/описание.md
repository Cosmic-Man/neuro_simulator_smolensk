ОПИСАНИЕ ПРОЕКТА NEURO_SIMULATOR_SMOLENSK
==========================================

1. Назначение проекта
---------------------

Проект предназначен для расчёта интегральных показателей качества городской
среды Смоленска. Исходные статистические показатели объединяются при помощи
нечёткой свёртки: числовые значения переводятся в лингвистические оценки,
обрабатываются набором экспертных правил, после чего преобразуются обратно в
числовой индекс качества.

Основная работа выполняется в Jupyter Notebook. Проект носит исследовательский
и демонстрационный характер: модели, диапазоны функций принадлежности и правила
задаются непосредственно в ячейках ноутбука.

5. Базовые компоненты нечёткой логики
-------------------------------------

5.1. Класс LinguisticVariable

Класс описывает одну лингвистическую переменную. Для неё задаются:

    name          — отображаемое имя;
    universe_min  — нижняя граница числового диапазона;
    universe_max  — верхняя граница диапазона;
    terms         — словарь лингвистических термов.

Терм состоит из имени, типа функции принадлежности и её параметров. Класс
поддерживает фаззификацию отдельного значения, получение всех степеней
принадлежности и построение графиков.

5.2. Функции принадлежности

trimf(x, a, b, c)
    Треугольная функция принадлежности. Значение растёт от a до b и уменьшается
    от b до c.

trapmf(x, a, b, c, d)
    Трапециевидная функция принадлежности. Между b и c степень принадлежности
    равна единице.

5.3. Класс FuzzySystem

Класс хранит входные переменные, выходную переменную и экспертные правила.
Основные методы:

    add_input()   — добавление входной переменной;
    set_output()  — назначение выходной переменной;
    add_rule()    — добавление правила;
    compute()     — расчёт итогового индекса;
    _defuzzify()  — преобразование нечёткого результата в число.

5.4. Последовательность вычисления

    1. Фаззификация входных значений.
    2. Проверка предпосылок каждого правила.
    3. Расчёт силы правила как минимума степеней принадлежности.
    4. Агрегация результатов по максимуму.
    5. Дефаззификация методом центра тяжести по сетке из 1000 точек.

Функция apply_fuzzy_convolution(df) применяет активную нечёткую систему к
каждой строке подготовленной таблицы и возвращает массив итоговых индексов.

5.5. Что такое нечёткое множество

В классической логике объект либо принадлежит множеству, либо не принадлежит
ему. Например, при жёстком пороге можно считать исполнение бюджета не ниже 80 %
«хорошим», а 79,9 % — уже «нехорошим». Такое решение создаёт резкий переход,
хотя практически эти значения почти одинаковы.

Нечёткое множество задаёт не бинарный признак, а степень принадлежности:

    μ_A(x) ∈ [0, 1],

где x — исходное числовое значение, A — качественное понятие, а μ_A(x) —
степень соответствия значения этому понятию. Значение 0 означает отсутствие
принадлежности, 1 — полное соответствие, промежуточные числа — частичное
соответствие.

Одно число может одновременно принадлежать нескольким соседним термам.
Например, исполнение бюджета 82 % может иметь принадлежность 0,4 к терму
«среднее» и 0,6 к терму «высокое». Это не вероятность: сумма степеней
принадлежности не обязана равняться единице. Степень показывает близость к
смыслу терма, а не вероятность наступления события.

5.6. Что такое лингвистическая переменная

Лингвистическая переменная — числовой показатель, значения которого
интерпретируются словами естественного языка. Она состоит из:

    имени — например, «Исполнение бюджета»;
    универсального множества — допустимого диапазона, например [60, 100];
    набора термов — «очень низкое», «низкое», «среднее», «высокое»;
    функций принадлежности — правил перевода числа в каждый терм.

В коде этим понятиям соответствуют объект LinguisticVariable, поля
universe_min и universe_max, словарь terms и метод fuzzify().

5.7. Треугольная функция принадлежности

Треугольная функция задаётся тремя параметрами a, b и c:

    μ(x) = 0,                  если x ≤ a или x ≥ c;
    μ(x) = (x - a)/(b - a),   если a < x ≤ b;
    μ(x) = (c - x)/(c - b),   если b < x < c.

Параметры имеют следующий смысл:

    a — левая граница, за которой терм не проявляется;
    b — центральное значение с принадлежностью 1;
    c — правая граница, за которой терм не проявляется.

Например, trimf(82, 72, 78, 84) описывает принадлежность значения 82 терму со
средним центром 78. Поскольку 82 находится на убывающей стороне треугольника,
степень принадлежности равна (84 - 82)/(84 - 78) = 1/3.

Треугольная функция подходит для понятий с одним наиболее характерным
значением: «среднее», «около 80 %», «типичный уровень».

5.8. Трапециевидная функция принадлежности

Трапециевидная функция задаётся четырьмя параметрами a, b, c и d:

    μ(x) = 0,                  если x ≤ a или x ≥ d;
    μ(x) = (x - a)/(b - a),   если a < x ≤ b;
    μ(x) = 1,                  если b < x ≤ c;
    μ(x) = (d - x)/(d - c),   если c < x < d.

Отрезок [b, c] образует плато: все значения внутри него считаются полностью
соответствующими терму. В отличие от треугольной функции, здесь максимум не
сосредоточен в одной точке.

Трапециевидная функция удобна для нормативных диапазонов. Например, если все
значения от 80 до 90 должны одинаково считаться «хорошими», нет необходимости
выбирать единственный идеальный центр.

5.9. Фаззификация

Фаззификация — перевод конкретного числового наблюдения в набор степеней
принадлежности. Метод get_all_memberships(x) проходит по всем термам входной
переменной и формирует словарь вида:

    {
        "низкое": 0.0,
        "среднее": 0.33,
        "высокое": 0.67,
        "очень высокое": 0.0
    }

Для одной строки DataFrame метод FuzzySystem.compute() создаёт такой словарь
для каждого входа. Результат фаззификации хранится в переменной memberships и
служит входом для базы правил.

5.10. Нечёткое правило и операция «И»

Правило состоит из предпосылки premise и заключения consequent. В коде оно
представлено словарём:

    {
        "premise": {
            "budget": "высокое",
            "yards": "много",
            "satisfaction": "высокая"
        },
        "consequent": "отлично"
    }

Логическое «И» реализовано оператором минимума. Если степени принадлежности
трёх условий равны 0,8, 0,6 и 0,7, сила срабатывания правила равна:

    α = min(0,8; 0,6; 0,7) = 0,6.

Такой подход соответствует нечёткому выводу Мамдани. Смысл минимума состоит в
том, что правило не может сработать сильнее самого слабого своего условия.

5.11. Агрегация заключений

Несколько правил могут приводить к одному выходному терму. Например, разные
сочетания показателей могут дать результат «хорошо» с силами 0,3, 0,7 и 0,5.
В реализации сохраняется максимальная сила:

    α_«хорошо» = max(0,3; 0,7; 0,5) = 0,7.

После обработки всех правил получается словарь aggregated, содержащий итоговую
активацию каждого выходного терма. При построении результирующей нечёткой
фигуры функция принадлежности каждого терма ограничивается сверху его силой
активации:

    μ'_term(z) = min(α_term, μ_term(z)).

Затем усечённые фигуры всех выходных термов объединяются оператором максимума:

    μ_result(z) = max_term μ'_term(z).

5.12. Дефаззификация

Дефаззификация превращает объединённую нечёткую фигуру в одно число. В проекте
используется дискретный вариант метода центра тяжести:

    result = Σ(z_i · μ_result(z_i)) / Σ μ_result(z_i).

Диапазон выходной переменной делится на 1000 равномерных точек z_i. Для каждой
точки вычисляется итоговая степень принадлежности, после чего находится
взвешенный центр всей фигуры. Чем сильнее активированы высокие выходные термы,
тем правее смещается центр тяжести и тем выше итоговый индекс.

Если ни одно правило не активировалось и знаменатель равен нулю, реализация
возвращает 0. Это специальное техническое значение, которое означает отсутствие
подходящего вывода, а не обязательно реальное нулевое качество объекта.

5.13. Роль функции apply_fuzzy_convolution

DataFrame хранит несколько временных наблюдений, а FuzzySystem.compute()
рассчитывает только одно наблюдение. apply_fuzzy_convolution() соединяет эти
уровни:

    1. df.apply(..., axis=1) перебирает строки таблицы;
    2. row.to_dict() превращает строку в словарь «имя входа -> значение»;
    3. fuzzy_system.compute() вычисляет индекс для словаря;
    4. .values возвращает массив индексов в исходном порядке строк;
    5. массив присваивается новому столбцу соответствующей таблицы.

Короткие имена budget, roads, norm и другие являются интерфейсом между
подготовкой данных и нечёткой системой. Имя входа, указанное в add_input(),
должно точно совпадать с ключом в словаре строки.


6. Предметные блоки ноутбука
----------------------------

6.1. Современная городская среда

Входы модели:
    budget       — исполнение бюджета;
    yards        — количество благоустроенных дворов;
    satisfaction — удовлетворённость городской средой.

Результат:
    индекс качества современной городской среды.

6.2. Дорожно-транспортный комплекс

Использует показатели состояния дорог, пассажиропотока, расписания, скорости,
ДТП и устранения дефектов. Формирует интегральную оценку качества транспортной
системы. Внутри раздела также создаются более узкие выборки для отдельных
вариантов расчёта.

6.3. Доступная среда

Входы модели:
    budget  — исполнение бюджета;
    events  — количество завершённых мероприятий;
    support — количество получателей адресной поддержки.

Результат:
    индекс качества доступной среды.

6.4. Общественные пространства

Входы модели:
    budget       — исполнение бюджета;
    territory    — благоустроенные общественные территории;
    satisfaction — удовлетворённость городской средой.

Результат:
    индекс качества общественных пространств.

6.5. Городской общественный транспорт

Использует отдельный набор транспортных показателей и правил. Оценивает работу
городского транспорта с учётом пассажиропотока, расписания, скорости,
безопасности и состояния инфраструктуры.

6.6. Парковки и безопасность движения

Входы модели:
    budget — исполнение бюджета;
    roads  — объём отремонтированных дорог;
    norm   — доля дорог в нормативном состоянии;
    defect — срок устранения дефектов.

Результат:
    индекс качества парковок и безопасности движения.

6.7. Внутренняя структура предметного блока

Несмотря на различие показателей и правил, каждый предметный раздел построен по
одной схеме:

    1. Из общего df выбирается диапазон столбцов или их именованный набор.
    2. Исходные длинные русские названия заменяются короткими техническими
       именами.
    3. Через describe() проверяются количество значений, среднее, стандартное
       отклонение, минимум, квартили и максимум.
    4. Для каждого входа создаётся объект LinguisticVariable.
    5. В объект добавляются термы и параметры функций принадлежности.
    6. Создаётся выходная LinguisticVariable с качественными уровнями индекса.
    7. Создаётся новый FuzzySystem и регистрируются его входы и выход.
    8. Формируется список экспертных правил rules.
    9. Каждое правило добавляется в систему методом add_rule().
   10. apply_fuzzy_convolution() рассчитывает индекс для всех строк.
   11. Полученный массив записывается в новый столбец.
   12. Повторный describe() показывает распределение рассчитанного индекса.

Подготовительная часть принципиально важна. Нечёткая модель работает не с
позициями столбцов, а с именами входов. Например, после переименования модель
может запросить inputs_values['budget']. Если столбец не был переименован,
словарь строки не содержит такого ключа, и вычисление не может продолжиться.

6.8. Два уровня транспортных моделей

Разделы дорожного комплекса и городского общественного транспорта содержат не
одну, а несколько выборок:

    базовая таблица — полный набор показателей соответствующей подсистемы;
    r1 — расширенная модель с показателями дорог, пассажиропотока, расписания,
         скорости, ДТП и срока устранения дефектов;
    r2 — компактная модель благополучия дорог на основе нормативного состояния,
         бюджета и регулируемых переходов.

Такое разделение позволяет получать частные индексы. Один индекс отвечает на
вопрос о комплексном качестве транспортной системы, другой — о состоянии
дорожной инфраструктуры. Их не следует воспринимать как взаимозаменяемые:
наборы входов и смысл выходных оценок различаются.

6.9. Карта ячеек ноутбука

Ноутбук содержит 70 ячеек с индексами от 0 до 69. Логическая карта имеет вид:

    0–4   Общие классы, функции принадлежности и функция применения модели.
    5–7   Импорт библиотек, загрузка CSV и общая информация о DataFrame.
    8–17  Современная городская среда, расчёт и заготовки визуализаций.
    18–32 Дорожно-транспортный комплекс и две его подмодели.
    33–39 Доступная среда.
    40–46 Общественные пространства.
    47–61 Городской общественный транспорт и его подмодели.
    62–68 Парковки и безопасность движения.
    69    Пустая резервная ячейка.

Первые пять ячеек определяют общий вычислительный аппарат. Предметные разделы
не содержат собственных копий классов и используют ранее определённые объекты.
Каждый раз при переходе к новой модели переменная fuzzy_system получает новый
объект FuzzySystem. Поэтому смысл этой переменной определяется последней
выполненной ячейкой создания правил.

6.10. Смысл описательной статистики

Вызов describe() используется не только для отображения таблицы. Он позволяет
сопоставить функции принадлежности с реальным диапазоном данных:

    count — показывает наличие пропусков;
    mean — даёт представление о центральном уровне показателя;
    std — характеризует разброс;
    min и max — помогают назначить границы универсального множества;
    25 %, 50 % и 75 % — помогают расположить центры соседних термов.

Если параметры функции принадлежности находятся далеко от фактического
диапазона, большинство наблюдений получит нулевые степени принадлежности. Если
термы слишком широкие, разные состояния будут плохо различаться. Поэтому
статистика является связующим звеном между исходными данными и экспертной
настройкой модели.


7. Экспертные правила
---------------------

Для каждой подсистемы создаётся собственный объект FuzzySystem. Числовые
показатели разделяются на термы наподобие:

    очень низкое, низкое, среднее, высокое, очень высокое;
    очень мало, мало, средне, много, очень много;
    медленно, средне, быстро.

Выходной индекс также разбивается на качественные уровни, например:

    катастрофа, плохо, удовлетворительно, хорошо, отлично, превосходно.

Правила имеют форму:

    ЕСЛИ бюджет высокий
    И показатель инфраструктуры высокий
    И удовлетворённость высокая,
    ТО итоговая оценка отличная.

Наборы правил заданы явно в списках rules, после чего последовательно
добавляются в соответствующую систему.


8. Общий поток данных
---------------------

    CSV-файл
        -> pandas.DataFrame
        -> выбор столбцов предметного блока
        -> переименование в короткие внутренние имена
        -> фаззификация значений
        -> применение экспертных правил
        -> агрегация нечётких результатов
    -> дефаззификация
    -> числовой индекс качества
    -> таблицы, статистика и графики в JupyterLab.

8.1. Прохождение одного наблюдения

Каждая строка CSV соответствует одному временному периоду. Для примера можно
рассмотреть условную строку блока «Современная городская среда»:

    budget = 82;
    yards = 40;
    satisfaction = 47.

Сначала строка становится словарём:

    {
        "budget": 82,
        "yards": 40,
        "satisfaction": 47
    }

Затем каждое число независимо фаззифицируется. Условный результат может иметь
вид:

    budget:
        среднее = 0,33;
        высокое = 0,33.

    yards:
        средне = 0,29;
        много = 0,29.

    satisfaction:
        средняя = 0;
        высокая = 0,62.

После этого система проверяет все правила. Правило с несовместимым термом
получает силу 0. Правило, для которого все термы активны, получает минимальную
из их степеней. Поэтому одно наблюдение обычно активирует сразу несколько
соседних правил с разной силой.

Силы правил группируются по выходным термам. Например:

    плохо = 0;
    удовлетворительно = 0,29;
    хорошо = 0,33;
    отлично = 0,10.

Эти значения обрезают функции принадлежности выходных термов, после чего центр
тяжести объединённой фигуры даёт конкретное число, например 65,4. Число 65,4
не выбирается отдельным правилом: оно возникает как компромисс между всеми
активированными заключениями.

Приведённые степени и итоговое число являются иллюстрацией механизма. Точные
значения определяются параметрами функций и базой правил конкретного раздела.

8.2. Почему нечёткая модель даёт плавный результат

Если исходный показатель изменится с 82 до 82,1, степени принадлежности также
изменятся немного. Силы правил и центр тяжести сдвинутся плавно. Это отличает
нечёткую модель от системы жёстких порогов, где переход через одну границу
может мгновенно изменить категорию целиком.

Плавность полезна для городских индикаторов, поскольку измерения содержат шум,
округления и естественные колебания. Близкие по смыслу периоды должны получать
близкие оценки, даже если один из показателей оказался по разные стороны
условного порога.

8.3. Как читать итоговый индекс

Итоговый индекс — агрегированная оценка набора входных показателей в шкале
соответствующей выходной LinguisticVariable. Он отражает одновременно:

    положение каждого входного значения относительно его термов;
    сочетание входов, описанное экспертными правилами;
    силу срабатывания каждого правила;
    форму и расположение выходных функций принадлежности.

Индекс не является простым средним входных столбцов. Два периода с одинаковым
средним значением показателей могут получить разные результаты, если один из
них содержит критически низкий показатель. Оператор min уменьшит силу хорошего
правила до уровня самого слабого условия.

Также индекс не следует интерпретировать как процент вероятности или точную
физическую величину. Это безразмерная экспертная оценка на заданной шкале. Её
смысл определяется названиями и параметрами выходных термов.

8.4. Временная интерпретация данных

Столбцы period_start, year, quarter и period_index не входят непосредственно в
нечёткие правила. Они идентифицируют наблюдение во времени и позволяют после
расчёта:

    строить динамику индекса по кварталам;
    сравнивать одинаковые кварталы разных лет;
    искать периоды роста и снижения;
    сопоставлять изменение интегрального индекса с отдельными показателями.

Расчёт каждой строки независим: модель не использует предыдущие значения и не
строит прогноз. Поэтому текущая реализация является системой многокритериальной
оценки временных наблюдений, а не нейронной сетью и не моделью временного ряда.

8.5. Что в проекте называется нечёткой свёрткой

Под свёрткой здесь понимается объединение нескольких разнородных критериев в
один интегральный показатель. Слово «нечёткая» указывает, что объединение
выполняется не фиксированной линейной формулой, а через:

    лингвистические термы;
    функции принадлежности;
    экспертные правила;
    нечёткие операции min и max;
    дефаззификацию.

Это понятие не связано со свёрточными нейронными сетями и операцией convolution
из обработки изображений. В проекте отсутствуют обучение весов, обратное
распространение ошибки и нейроны. Все параметры задаются человеком.

8.6. Источник поведения модели

Поведение модели определяется четырьмя уровнями настройки:

    данные — фактические значения городских показателей;
    функции принадлежности — смысл слов «низко», «средне», «высоко»;
    правила — допустимые сочетания причин и результатов;
    выходная шкала — перевод качественных заключений в итоговое число.

Изменение любого уровня может изменить результат. Например, расширение терма
«высокое исполнение бюджета» увеличит число периодов, которые активируют
соответствующие правила. Изменение одного заключения с «хорошо» на «отлично»
сдвинет центр тяжести вверх для всех строк, где это правило срабатывает.

8.7. Полнота базы правил

Если каждый вход имеет по пять термов, полная база для трёх входов теоретически
содержит 5 × 5 × 5 = 125 комбинаций. Для шести входов с тремя термами полная
база уже содержит 3⁶ = 729 комбинаций. Поэтому число правил быстро растёт с
числом входов.

Полная база обеспечивает заключение для любой комбинации ненулевых термов.
Неполная база допустима, если исключённые сочетания невозможны или не имеют
смысла, однако при отсутствии активных правил система возвращает 0. Таким
образом, покрытие пространства входов — важная характеристика модели.

8.8. Перекрытие термов

Соседние функции принадлежности намеренно перекрываются. В области перекрытия
значение одновременно частично относится к двум термам, благодаря чему
переходы между качественными состояниями становятся плавными.

Недостаточное перекрытие создаёт разрывы, где все степени близки к нулю.
Чрезмерное перекрытие делает термы плохо различимыми и активирует слишком много
правил одновременно. Настройка пересечений должна сохранять баланс между
устойчивостью и различимостью состояний.

8.9. Термины проекта

Нечёткое множество
    Множество, принадлежность к которому выражается числом от 0 до 1.

Степень принадлежности
    Мера соответствия числового значения качественному понятию.

Лингвистическая переменная
    Числовой показатель, описываемый набором словесных термов.

Терм
    Отдельное качественное значение лингвистической переменной, например
    «низкое» или «высокое».

Универсальное множество
    Числовой диапазон, на котором определена переменная.

Функция принадлежности
    Функция, переводящая число в степень принадлежности определённому терму.

Фаззификация
    Перевод точных входных чисел в нечёткие степени принадлежности.

Предпосылка правила
    Набор условий в части «ЕСЛИ».

Заключение правила
    Выходной терм в части «ТО».

Сила правила
    Число от 0 до 1, показывающее, насколько правило применимо к наблюдению.

Агрегация
    Объединение результатов нескольких правил в одну выходную нечёткую фигуру.

Дефаззификация
    Перевод выходной нечёткой фигуры в конкретное числовое значение.

Интегральный индекс
    Итоговая числовая оценка, объединяющая несколько исходных критериев.

Система Мамдани
    Схема нечёткого вывода, в которой предпосылки и заключения представлены
    нечёткими множествами, а результат получается через агрегацию и
    дефаззификацию.
