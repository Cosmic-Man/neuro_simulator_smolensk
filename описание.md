ОПИСАНИЕ ПРОЕКТА NEURO_SIMULATOR_SMOLENSK
==========================================

1. Назначение проекта
---------------------

Проект предназначен для расчёта интегральных показателей качества городской
среды Смоленска. Исходные статистические показатели объединяются при помощи
нечёткой свёртки: числовые значения переводятся в лингвистические оценки,
обрабатываются набором экспертных правил, после чего преобразуются обратно в
числовой индекс качества.

Основная работа выполняется в Jupyter Notebook. Проект носит исследовательский
и демонстрационный характер: модели, диапазоны функций принадлежности и правила
задаются непосредственно в ячейках ноутбука.

4. Структура исходного датасета
-------------------------------

Первые четыре столбца описывают период наблюдения:

    0  period_start — дата начала периода;
    1  year — год;
    2  quarter — квартал;
    3  period_index — последовательный номер периода.

Остальные столбцы сгруппированы по городским подсистемам:

    4–6   Современная городская среда:
          исполнение бюджета, благоустроенные дворы,
          удовлетворённость городской средой.

    7–15  Дорожно-транспортный комплекс:
          исполнение бюджета, отремонтированные дороги, нормативное состояние
          дорог, пассажиропоток, соблюдение расписания, средняя скорость,
          количество ДТП, срок устранения дефектов и регулируемые переходы.

    16–18 Доступная среда:
          исполнение бюджета, завершённые мероприятия,
          получатели адресной поддержки.

    19–21 Общественные пространства:
          исполнение бюджета, благоустроенные общественные территории,
          удовлетворённость городской средой.

    22–30 Городской общественный транспорт:
          повторный набор транспортных показателей для отдельной модели.

    31–34 Парковки и безопасность движения:
          исполнение бюджета, отремонтированные дороги,
          нормативное состояние дорог и срок устранения дефектов.

При чтении данных выполняется очистка пробелов в названиях столбцов:

    df.columns = df.columns.str.strip()

Это важно, поскольку первый показатель бюджета в CSV содержит завершающий
пробел.


5. Базовые компоненты нечёткой логики
-------------------------------------

5.1. Класс LinguisticVariable

Класс описывает одну лингвистическую переменную. Для неё задаются:

    name          — отображаемое имя;
    universe_min  — нижняя граница числового диапазона;
    universe_max  — верхняя граница диапазона;
    terms         — словарь лингвистических термов.

Терм состоит из имени, типа функции принадлежности и её параметров. Класс
поддерживает фаззификацию отдельного значения, получение всех степеней
принадлежности и построение графиков.

5.2. Функции принадлежности

trimf(x, a, b, c)
    Треугольная функция принадлежности. Значение растёт от a до b и уменьшается
    от b до c.

trapmf(x, a, b, c, d)
    Трапециевидная функция принадлежности. Между b и c степень принадлежности
    равна единице.

5.3. Класс FuzzySystem

Класс хранит входные переменные, выходную переменную и экспертные правила.
Основные методы:

    add_input()   — добавление входной переменной;
    set_output()  — назначение выходной переменной;
    add_rule()    — добавление правила;
    compute()     — расчёт итогового индекса;
    _defuzzify()  — преобразование нечёткого результата в число.

5.4. Последовательность вычисления

    1. Фаззификация входных значений.
    2. Проверка предпосылок каждого правила.
    3. Расчёт силы правила как минимума степеней принадлежности.
    4. Агрегация результатов по максимуму.
    5. Дефаззификация методом центра тяжести по сетке из 1000 точек.

Функция apply_fuzzy_convolution(df) применяет активную нечёткую систему к
каждой строке подготовленной таблицы и возвращает массив итоговых индексов.


6. Предметные блоки ноутбука
----------------------------

6.1. Современная городская среда

Входы модели:
    budget       — исполнение бюджета;
    yards        — количество благоустроенных дворов;
    satisfaction — удовлетворённость городской средой.

Результат:
    индекс качества современной городской среды.

6.2. Дорожно-транспортный комплекс

Использует показатели состояния дорог, пассажиропотока, расписания, скорости,
ДТП и устранения дефектов. Формирует интегральную оценку качества транспортной
системы. Внутри раздела также создаются более узкие выборки для отдельных
вариантов расчёта.

6.3. Доступная среда

Входы модели:
    budget  — исполнение бюджета;
    events  — количество завершённых мероприятий;
    support — количество получателей адресной поддержки.

Результат:
    индекс качества доступной среды.

6.4. Общественные пространства

Входы модели:
    budget       — исполнение бюджета;
    territory    — благоустроенные общественные территории;
    satisfaction — удовлетворённость городской средой.

Результат:
    индекс качества общественных пространств.

6.5. Городской общественный транспорт

Использует отдельный набор транспортных показателей и правил. Оценивает работу
городского транспорта с учётом пассажиропотока, расписания, скорости,
безопасности и состояния инфраструктуры.

6.6. Парковки и безопасность движения

Входы модели:
    budget — исполнение бюджета;
    roads  — объём отремонтированных дорог;
    norm   — доля дорог в нормативном состоянии;
    defect — срок устранения дефектов.

Результат:
    индекс качества парковок и безопасности движения.


7. Экспертные правила
---------------------

Для каждой подсистемы создаётся собственный объект FuzzySystem. Числовые
показатели разделяются на термы наподобие:

    очень низкое, низкое, среднее, высокое, очень высокое;
    очень мало, мало, средне, много, очень много;
    медленно, средне, быстро.

Выходной индекс также разбивается на качественные уровни, например:

    катастрофа, плохо, удовлетворительно, хорошо, отлично, превосходно.

Правила имеют форму:

    ЕСЛИ бюджет высокий
    И показатель инфраструктуры высокий
    И удовлетворённость высокая,
    ТО итоговая оценка отличная.

Наборы правил заданы явно в списках rules, после чего последовательно
добавляются в соответствующую систему.


8. Порядок локального запуска
-----------------------------

Из корня проекта выполнить в PowerShell:

    python -m venv .venv
    .\.venv\Scripts\Activate.ps1
    python -m pip install --upgrade pip
    python -m pip install -r requirements.txt
    python -m jupyter lab --ServerApp.ip=127.0.0.1 --ServerApp.port=8888

Затем открыть в браузере:

    http://localhost:8888/

Привязка к 127.0.0.1 оставляет Jupyter доступным только на текущем компьютере.
Не следует использовать 0.0.0.0 или *, особенно в публичной Wi-Fi-сети.

Перед полным выполнением ноутбука необходимо убедиться, что файл
smolensk_dataset (1).csv находится в корне проекта. После замены данных или
изменения ячеек рекомендуется выбрать Kernel -> Restart Kernel, а затем
Run -> Run All Cells.


9. Важные особенности и ограничения
------------------------------------

1. Переменная fuzzy_system глобальная и переопределяется в каждом предметном
   блоке. Ячейки необходимо запускать сверху вниз. Запуск отдельной старой
   ячейки после выполнения следующего раздела может применить к таблице чужую
   модель и вызвать KeyError.

2. Структура ноутбука зависит от порядка столбцов CSV. Выборки создаются через
   df.iloc, поэтому добавление, удаление или перестановка столбцов требует
   обновления соответствующих диапазонов.

3. Названия столбцов должны точно соответствовать ожидаемым строкам. Даже
   различия в точке, запятой, пробеле или суффиксе могут помешать rename() и
   привести к отсутствию внутренних имён budget, roads, norm и других.

4. Диапазоны функций принадлежности и правила являются экспертными. Перед
   использованием результатов для практических решений их необходимо
   валидировать на реальных данных и согласовать с предметными специалистами.

5. CSV и XLSX не хранятся в Git. После клонирования репозитория набор данных
   нужно получить отдельно и поместить рядом с ноутбуком.

6. Выводы Jupyter могут значительно увеличивать размер файла и замедлять работу
   браузера. Перед коммитом полезно очищать тяжёлые табличные выводы через
   Restart Kernel and Clear Outputs.
