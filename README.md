# Нейросимулятор Смоленска

Локальный демонстрационный прототип для анализа городских показателей, сценарного моделирования FCM × ANFIS и подготовки рекомендаций заказчику. Основа расчётов — `primer/Pipeline (2).ipynb` и JSON-правила из папки `ruls`.

## Нейросимулятор Смоленска

Проект представляет собой локальное FastAPI-приложение для анализа городской мобильности и проверки управленческих сценариев. Приложение читает квартальные данные из XLSX, рассчитывает нечёткие индексы и прогнозы, показывает результаты в браузере и предоставляет API для тех же расчётов.

Основные пункты проекта:

- состояние показателей на последний квартал;
- история показателей и сезонность по кварталам;
- нечёткие и экспертные индексы, включая итоговый индекс Pipeline;
- сравнение фактических значений, baseline-моделей и ANFIS-прогнозов;
- нечёткая когнитивная карта FCM с экспертными и адаптированными весами;
- FCM-советчик для выбора проблемы и оценки возможных изменений;
- прогноз индексов по готовому или пользовательскому сценарию на выбранный горизонт;
- графики сценарного прогноза, ожидаемый эффект и интерпретация результата;
- анализ чувствительности факторов и оценка их влияния на выбранную цель;
- анализ данных Pipeline: диаграммы размаха, функции принадлежности и веса индексов;
- рекомендации заказчику, приоритизация направлений и объяснение результата сценария;
- загрузка, выбор, редактирование и выгрузка XLSX-датасетов;
- добавление нового квартала и переобучение моделей после подтверждения данных;
- сохранение, загрузка и выгрузка пользовательских сценариев в формате JSON.

Проект намеренно упрощён:

- нет авторизации, пользователей и ролей;
- нет PostgreSQL, SQLAlchemy и миграций;
- нет Docker и Docker Compose;
- исходные и новые квартальные данные хранятся в XLSX-файлах папки `datasets_ready`;
- пользовательский сценарий можно загрузить, скачать и сохранить как JSON в `runtime/scenarios`.

## Запуск в Windows PowerShell

Откройте PowerShell в корневой папке проекта и выполните:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

Откройте `http://127.0.0.1:8000`.

Повторный запуск:

```powershell
.\.venv\Scripts\Activate.ps1
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

## Работа с XLSX

Интерфейс показывает файлы `*.xlsx` из `datasets_ready`. Можно выбрать активный файл, изменить существующий квартал или добавить следующую строку. Изменения записываются прямо в выбранный XLSX.

После внесения подтверждённых данных нажмите «Переобучить модель». Нулевая служебная строка не принимается как фактический квартал. Кэш модели сохраняется локально в `runtime/models` и может быть пересоздан из XLSX.

Перед обновлением приложения сделайте резервную копию рабочего XLSX. Код можно заменить отдельно, после чего вернуть файл в `datasets_ready`.

## Сценарии

Готовые сценарии доступны сразу. Воздействия ограничены диапазоном `[-1, 1]`. Кнопка «Сохранить JSON» записывает проверенный сценарий в `runtime/scenarios/<id>.json`; сохранённые файлы загружаются при следующем запуске. Папка разрешена в `.gitignore`, поэтому нужные JSON можно коммитить и передавать вместе с проектом. Кнопки «Выгрузить JSON» и «Загрузить JSON» используются для ручного обмена файлами.
