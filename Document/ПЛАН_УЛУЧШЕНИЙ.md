# План улучшений локальной XLSX-версии

## Принятое решение

Проект работает как локальный демонстрационный стенд без авторизации, базы данных и контейнеризации. Единственный постоянный источник пользовательских числовых данных — XLSX в `datasets_ready`.

## Реализовано

- [x] удалены аккаунты, роли, сессии, CSRF и административный экран;
- [x] удалены PostgreSQL, SQLAlchemy, Alembic и серверное хранение сценариев;
- [x] удалены Dockerfile и Docker Compose;
- [x] сохранены Pipeline, FCM × ANFIS, бизнес-рекомендации и оценка эффекта;
- [x] добавлены выбор XLSX, редактирование квартала и новая строка;
- [x] добавлены переобучение и отображение состояния модели;
- [x] пользовательские сценарии импортируются и экспортируются как локальные JSON.

## Порядок работы заказчика

1. Выбрать проблемную бизнес-цель, например ДТП.
2. Получить пять приоритетных мер и проверить их в лаборатории.
3. Сравнить ожидаемый эффект и бюджетное направление.
4. Через квартал внести фактическую строку в активный XLSX.
5. Переобучить модель и повторить анализ.
