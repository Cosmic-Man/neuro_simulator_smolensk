# Нейросимулятор Смоленска — проблема Б

Локальный аналитический прототип моделирует транспортную доступность и безопасность городской мобильности. Проект объединяет новый сводный датасет, две нечёткие свёртки из Colab, FCM, ANFIS и классические baseline-модели.

## Данные и расчётный pipeline

Источник: `datasets_ready/smolensk_dataset_shared.xlsx`, лист `Лист1`.

- 80 кварталов: 2006Q1–2025Q4;
- 5 служебных полей и 31 числовой показатель;
- двухуровневый заголовок с шестью тематическими блоками;
- пустой исходный target заменяется воспроизводимыми расчётными целями.

Pipeline:

```text
Excel → проверка и train-only preprocessing
      → 8 нечётких индексов Кирилла
      → иерархический индекс Гульдар по 8 нечётким индексам
      → безопасность / регулярность / доступность
      → Seasonal Naive / Ridge / FCM / ANFIS
      → итоговый индекс и сценарный прогноз
```

Разбиение по времени выполняется без перемешивания:

- train: 2006Q1–2018Q4;
- validation: 2019Q1–2022Q4;
- test: 2023Q1–2025Q4.

Параметры нормализации, winsorization и моделей определяются без использования test.

## Colab-методы

Из notebook Кирилла перенесены `LinguisticVariable`, треугольные и трапециевидные функции принадлежности, Mamdani-агрегация, centroid-дефаззификация и все восемь таблиц правил — 2 164 правила суммарно.

Рассчитываются индексы современной городской среды, доступной среды, общественных пространств, парковок и безопасности, а также качество и благополучие дорог для двух транспортных блоков.

Из `idex_target.ipynb` перенесена экспертная свёртка: восемь нечётких индексов агрегируются в иерархический индекс с весами `0.6 / 0.6 / 0.7 / 0.2 / 0.5 / 0.7 / 0.9 / 0.4`. Входы нормализуются только по train. Иерархический индекс используется как независимый baseline и не заменяет три целевых ряда и согласованный итоговый индекс мобильности.

## Цели и модели

Три основные цели:

- безопасность движения — обратный нормализованный уровень ДТП;
- регулярность — средний показатель рейсов по расписанию;
- доступность — 30% регулярность + 20% скорость + 20% состояние дорог + 15% переходы + 15% нечёткая транспортная среда.



Сравниваются Seasonal Naive, Ridge по четырём лагам, экспертная FCM, адаптированная FCM и ANFIS. Метрики: MAE, RMSE, sMAPE, MASE и directional accuracy.

FCM содержит 16 узлов и 34 направленные связи. Обновление использует `alpha=0.35`, `lambda=1.3`; адаптированные веса рассчитываются как `0.70 × expert + 0.30 × data` с сохранением экспертного знака.

## Сценарии JSON

Встроены инерционный сценарий, ограниченный бюджет, ухудшение дорог, приоритет общественного транспорта, цифровая мобильность и повышение безопасности. Прямого ряда цифровизации нет, поэтому он явно обозначен как прокси регулярности, скорости и загруженности.

Пользовательский сценарий:

```json
{
  "version": 1,
  "id": "limited-budget-custom",
  "label": "Ограниченный бюджет",
  "description": "Пользовательская проверка",
  "mode": "adapted",
  "horizon": 8,
  "impulses": {
    "road_budget_execution": -0.15,
    "road_repair": -0.10
  }
}
```

Значения импульсов ограничены диапазоном `[-0.30, 0.30]`. Загруженные сценарии сохраняются локально в `runtime/scenarios/`, переживают перезапуск и не попадают в Git. Встроенные сценарии нельзя перезаписать.

API:

- `GET /api/metadata`, `/api/history`, `/api/indices`, `/api/evaluation`;
- `GET /api/fcm?mode=adapted`;
- `GET/POST /api/scenarios`;
- `POST /api/simulate`.

## Запуск

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Или:

```powershell
.\setup_local.ps1
.\run_local.ps1
```

Интерфейс: <http://127.0.0.1:8000>. API: <http://127.0.0.1:8000/docs>.
